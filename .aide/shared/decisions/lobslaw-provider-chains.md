# lobslaw-provider-chains

**Decision:** Provider chains with ordered steps: primary provider executes, reviewer provider critiques/synthesizes. Trigger conditions: min_complexity (1-10 scale), domain tags, or always. Resolver evaluates triggers in order, picks first match

## Rationale

Challenger model (MiniMax → Opus review) enables quality at speed. Chain is a first-class concept, not a workaround — resolver picks chain based on real-time task analysis
